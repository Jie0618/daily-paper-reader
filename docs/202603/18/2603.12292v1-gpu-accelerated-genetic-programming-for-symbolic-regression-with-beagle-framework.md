## 摘要
Beagle是一个新型软件框架，支持在GPU上执行遗传规划任务。该框架目前可用于符号回归，它通过最大化现有GPU平台吞吐量的方式来处理种群个体和训练适应度用例。在本研究中，我们报告了Beagle在Feynman符号回归数据集上的基准测试结果，并在相同的墙钟时间预算下，将其性能与名为StackGP的高速CPU系统以及广泛使用的PySR系统进行了比较。我们还报告了两种不同适应度函数的使用情况：一种是点对点误差函数，另一种是相关性适应度函数。结果表明，Beagle的GPU辅助符号回归性能显著优于领先的基于CPU的框架。

## Abstract
Beagle is a new software framework that enables execution of Genetic Programming tasks on the GPU. Currently available for symbolic regression, it processes individuals of the population and fitness cases for training in a way that maximizes throughput on extant GPU platforms. In this contribution, we report on the benchmarking of Beagle on the Feynman Symbolic Regression dataset and compare its performance with a fast CPU system called StackGP and the widely available PySR system under the same wall clock budget. We also report on the use of two different fitness functions, one a point-to-point error function, the other a correlation fitness function. The results demonstrate that the Beagle's GPU-aided Symbolic Regression significantly outperforms leading CPU-based frameworks.

### 2. 方法论：核心思想与关键技术
Beagle 框架采用了针对现代硬件深度优化的设计：
*   **异构分配**：GPU 负责演化运行和适应度函数评估（高并行任务）；CPU 负责选择、个体出生/死亡循环及变异（逻辑复杂任务）。
*   **ILGPU 与 C# 实现**：使用 ILGPU 库在底层直接操作 CUDA，避免了高级库的开销，并利用 C# 的高性能特性。
*   **内存管理优化**：引入“死亡池（Dead Pool）”机制，回收废弃个体的内存而非释放，几乎消除了垃圾回收（GC）和内存碎片带来的性能损耗。

*   **GCL 语言**：采用基于逆波兰表示法（RPN）的线性遗传规划语言（GCL），对 GPU 友好且不依赖堆内存。
*   **蒙特卡洛排名选择**：针对数百万种群，不再进行全量排序，而是通过对 100 个随机样本进行采样来估算百分位数，将选择效率提升了约 30,000 倍。
*   **适应度函数**：支持点对点误差（类似 RMSE）和基于相关性的适应度函数（$r^4$ 形式），并能通过识别 NaN（非数字）的位置来利用无效定义域的信息。

*   **充分性**：实验覆盖了 10 分钟和 30 分钟两个维度，并对 Beagle 内部的不同适应度函数进行了消融对比。

### 6. 主要结论与发现
*   **性能领先**：在 10 分钟和 30 分钟的测试中，Beagle（相关性适应度版本）的表现均显著优于 PySR 和 StackGP。
*   **相关性函数的优势**：使用相关性适应度函数（Correlation Fitness）虽然在 GPU 上计算更慢，但其搜索效率更高，最终解决的问题数量（典型成功率 82-84%）远高于点对点误差函数。
*   **NaN 处理的价值**：实验证明，通过奖励在相同位置产生 NaN 的个体，Beagle 能解决其他系统无法处理的具有特定定义域限制的方程（如二次公式）。
*   **规模效应**：Beagle 证明了在 GPU 加持下，维持数百万规模的种群是可行且高效的。

### 7. 优点（亮点）
*   **极致的工程优化**：从内存回收、线程调度到采样选择，全方位解决了 GP 在 GPU 上运行的瓶颈。
*   **处理无效值**：创新的 NaN 匹配机制让符号回归能够处理更复杂的数学定义域问题。
*   **实用性强**：强调短时间（<30 分钟）内的求解能力，更符合科研人员的实际使用习惯。

### 8. 不足与局限
*   **硬件依赖**：Beagle 的优势高度依赖 NVIDIA GPU，对于没有高性能显卡的用户无法体现价值。
*   **算力对比不均**：实验中 Beagle 使用了 2 个 V100 GPU，而对比系统仅使用 CPU，这虽然证明了 Beagle 对 GPU 利用的成功，但在算法逻辑层面的“公平性”比较略显薄弱。
*   **算子限制**：受限于 CUDA 块的线程限制，目前每个个体的适应度案例上限为 1024 个，对于超大规模数据集可能需要进一步的批处理优化。
*   **缺乏交叉算子**：当前版本的 Beagle 尚未实现交叉（Crossover）操作，仅依赖变异，这可能在某些极复杂问题的演化后期限制搜索能力。
